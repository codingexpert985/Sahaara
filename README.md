Sahaara

## AI-Powered Community Dispatch Platform

> **Describe a need. Create a mission. Find the right help. Move toward resolution.**

Sahaara is an AI-powered community dispatch application that transforms an unstructured community need into an actionable mission.

A user simply describes a problem in natural language. Sahaara uses AI to understand the request, identify its category and urgency, determine the resources needed, generate an action plan, and match the mission with suitable volunteers based on skill and resource coverage.

The entire journey is visualized through a **Relay Stepper**:

```text
Received
   ↓
AI Analysis
   ↓
Helpers Matched
   ↓
In Motion
   ↓
Resolved
The Problem

Community needs are often difficult to coordinate.

A person may know that they need help, but they may not know:

Who can help?
What resources are needed?
How urgent is the situation?
What steps should be taken?
How can progress be tracked?

At the same time, many people have the skills, resources, or willingness to help but do not know where their support is needed.

Existing platforms often focus on only one part of the problem:

Social platforms focus on communication.
Volunteer platforms focus on finding volunteers.
Donation platforms focus on financial contributions.
Chatbots provide information.

Sahaara combines understanding, planning, matching, and progress tracking into one workflow.

🎯 Core Features
1. Natural-Language Mission Creation

Users describe their community need in their own words.

Example:

"I need help finding blood for a patient in Karachi."

The system analyzes the request and creates a structured mission.

2. AI Problem Analysis

The AI identifies important information from the user's request:

Category
Urgency
Required Resources
Recommended Actions

This transforms a vague request into an actionable problem.

3. Action-Plan Generation

Sahaara automatically converts the problem into a list of tasks.

Example:

○ Identify required blood type
○ Find compatible donors
○ Contact available volunteers
○ Coordinate donation
○ Confirm completion

Each task can be checked off as the mission progresses.

4. Skill-Based Volunteer Matching

Sahaara matches missions with available volunteers using a coverage-based matching algorithm.

The matching system considers:

Volunteer skills
Available resources
Mission resource requirements
Whole-word token overlap

The algorithm avoids loose substring matching to prevent incorrect matches.

For example:

Mission Resource:
"blood donor"

Volunteer Skill:
"blood donation"

✓ Strong semantic token coverage

The system calculates a match score and ranks volunteers.

5. Interactive "Ask to Help" Flow

Users can interact with matched volunteers.

When a user clicks:

Ask to Help

the volunteer enters a requested state.

The mission relay advances to:

In Motion

This provides a clear visual indication that someone has begun taking action.

7. Task Progress Tracking

Each mission contains tasks generated by the AI.

Users can check tasks as completed.

Example:

🔌 API Endpoints
Create Mission
POST /api/missions

Creates a new AI-generated mission from a user's natural-language description.

Request
{
  "description": "A student needs a scientific calculator for an exam tomorrow."
}
Response

The backend returns a structured mission containing:

AI analysis
Category
Urgency
Required resources
Action-plan tasks
Matched volunteers
Update Task Status
PATCH /api/tasks/{task_id}/status

Updates the completion status of a mission task.

The frontend calls this endpoint when a user checks or unchecks a task.

Get Analytics
GET /api/analytics

Returns community impact statistics.

👥 Volunteer Matching

The current prototype includes an in-memory volunteer pool.

Example volunteer data includes:

Volunteer
├── Name
├── Skills
├── Resources
├── Availability
├── Location
└── Reliability

The matching system ranks volunteers based on how much of the mission's required resource profile they can cover.

The algorithm uses:

Whole-word token overlap with coverage-based scoring

This was specifically designed to avoid incorrect matches caused by loose substring matching.

🔐 Environment Variables

The Groq API key should be stored locally as an environment variable.

Windows PowerShell
$env:GROQ_API_KEY="your_api_key_here"

Then start the backend:

python -m uvicorn main:app --reload

⚠️ Never commit your API key to GitHub.

Do not place your API key directly inside:

main.py

Do not upload it to:

GitHub

If a key is accidentally exposed, revoke it and generate a new one.

⚙️ Backend Installation

Open a terminal inside the project folder:

cd backend

Create a virtual environment:

python -m venv venv

Activate it on Windows:

venv\Scripts\activate

Install dependencies:

pip install -r requirements.txt

Start the backend:

python -m uvicorn main:app --reload

The backend will be available at:

http://127.0.0.1:8000
 Frontend Installation

Open a second terminal:

cd frontend

Install dependencies:

npm install

If PowerShell blocks the npm command:

npm.cmd install

Start the development server:

npm.cmd run dev

The frontend will be available at:

http://localhost:5173

Current Data Architecture

Application Starts
       ↓
Volunteer Pool Loaded
       ↓

       ↓
AI Generates Mission
       ↓
Mission Exists in Memory
       ↓

       ↓

This approach was intentionally chosen for the hackathon MVP to keep the application lightweight and fast to develop.
